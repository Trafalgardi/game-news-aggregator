# Эксплуатация и диагностика

## Ежедневная работа

Штатно вмешательство не требуется. Workflow запускается два раза в сутки и сохраняет результаты в Git.

## Основные сигналы

### `status.json.error` растёт

Причины:

- сайт начал отдавать 403/429;
- RSS URL изменился;
- истёк timeout;
- DNS или TLS ошибка;
- HTML/sitemap больше не соответствует парсеру.

Действия:

1. открыть запись источника в `status.json`;
2. проверить сайт в браузере;
3. найти официальный RSS или sitemap;
4. прописать URL вручную;
5. временно отключить источник при постоянной блокировке.

### В `latest.json` мало материалов

Проверить:

- `window_hours`;
- даты публикаций;
- `max_age_days`;
- включён ли источник;
- не попала ли статья в дубль другого источника;
- не отсутствует ли дата в RSS/HTML.

### История быстро растёт

`articles.jsonl` хранит одну строку на canonical URL. Для нескольких лет объём останется приемлемым для Git. При достижении десятков мегабайт можно перенести старые записи в `data/archive/articles-YYYY.jsonl` и доработать загрузчик индекса canonical URL.

## Добавление источника

Добавить объект в `config/sources.yaml`, запустить:

```powershell
python -m game_news validate-config
python -m game_news collect --dry-run --source SOURCE_ID
```

После успешного dry-run включить источник и отправить изменения.

## Изменение приоритетов

`priority` должен находиться в диапазоне 0–100. Итоговый `importance_score` зависит от:

- приоритета источника;
- совпавших тематических ключей;
- тегов источника;
- уверенности в дате.

Это предварительная сортировка. Финальную важность определяет ChatGPT.

## Восстановление

Все состояния хранятся в Git. Для возврата:

```powershell
git log --oneline
git revert COMMIT_SHA
git push
```

Не удаляйте историю force-push без необходимости.
